Configuration (fptest.ini)

fpTest reads test project related information from the fptest.ini file, which will be auto-created in the same directory as the binary file.

Excluding tests

Individual tests can be excluded separately from Enabling and Disabling tests. Test state will be not cleared when globally enabling tests.

Excluding tests is allowed at multiple levels:

  • per test project,
  • per test suite,
  • per test case,
  • per test method.

Excluded tests are marked:

  • with a red X in treeview in GUI version,
  • with an x in place of dots in console mode (TextTestRunner).

Excluding test in GUI

To exclude a test in the GUI version right click on the test and choose Exclude Test Execution. To re-include tests right click on the test and choose Include Test Execution.

Excluding test in fptest.ini

Configuration file allows explicitly to disable the tests you don't want to run and fpTest will run all the others. This was done for a specific reason - when new tests are implemented, they will be automatically allowed. This is especially important when you do something like Test Driven Development (TDD).

Example:

  • Project1.exe (binary name)
    • Suite1 (test suite)
      • TClassA (test case)
        • TestOne (test method)
        • TestTwo (test method)
        • TestThree (test method)
      • TClassB (test case)
        • TestOne (test method)
        • TestTwo (test method)
        • TestThree (test method)

With an empty or non-existant fptest.ini file, all 6 test methods will run. Three from TClassA and three from TClassB.

Disabling test project
To disable test in whole project fptest.ini file will look like this:

[Excluded_Test]
project1.exe=0

Disabling test suite
If we want to disable Suite1 completely, fptest.ini file will look as follows:

[Excluded_Test.Project1.exe]
Suite1=0

Disabling test case
To disable TClassB completely:

[Excluded_Test.Project1.exe.Suite1]
TClassA=0

Note: you don't have to disable each test method one by one.

Disabling test in test case
To disable TestOne() and TestTwo() from the TClassB test suite fptest.ini file will look as follows:

[Excluded_Test.project1.exe.Suite1.TClassA]
TestOne=0
TestTwo=0
